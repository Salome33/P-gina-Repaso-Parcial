JAVA - Programación Orientada a Objetos

1. Paradigma de Clase (POO)

// Clase: plantilla para crear objetos public class Persona { // Atributos (características) String nombre; int edad; // Métodos (comportamientos) public void saludar() { System.out.println("Hola, soy " + nombre); } } // Crear objeto (instancia) Persona persona1 = new Persona(); persona1.nombre = "Juan"; persona1.edad = 25; persona1.saludar();
Concepto:
La POO organiza el código en clases (plantillas) y objetos (instancias).

Ventajas:
• Reutilización de código
• Modularidad
• Modelar entidades reales

2. Métodos y Atributos

public class Coche { // ATRIBUTOS (estado) String marca; String modelo; int velocidad; // MÉTODOS (comportamiento) public void acelerar(int incremento) { velocidad += incremento; System.out.println("Velocidad: " + velocidad + " km/h"); } public void frenar(int decremento) { velocidad -= decremento; if(velocidad < 0) velocidad = 0; } }
Atributos:
Variables que almacenan el estado del objeto

Métodos:
Funciones que definen el comportamiento

4. Encapsulamiento (Getters y Setters)

public class Estudiante { private String nombre; private int edad; private double promedio; // GETTER: obtener valor public String getNombre() { return nombre; } // SETTER: establecer con validación public void setNombre(String nombre) { if(nombre != null && !nombre.isEmpty()) { this.nombre = nombre; } } public void setEdad(int edad) { if(edad > 0 && edad < 150) { this.edad = edad; } } }
Protección de datos:

• Atributos privados
• Acceso controlado
• Validación de datos
• Mayor seguridad

5. Constructor y Sobrecarga

public class Producto { private String nombre; private double precio; private int stock; // Constructor por defecto public Producto() { this.nombre = "Sin nombre"; this.precio = 0.0; this.stock = 0; } // Constructor con parámetros public Producto(String nombre, double precio) { this.nombre = nombre; this.precio = precio; this.stock = 0; } // Constructor completo public Producto(String nombre, double precio, int stock) { this.nombre = nombre; this.precio = precio; this.stock = stock; } }
Constructores:

Inicializan objetos al crearlos

Sobrecarga:
Múltiples constructores con diferentes parámetros

6. Herencia y Agregación

Herencia

public class Persona { protected String nombre; protected int edad; public void presentarse() { System.out.println(nombre); } } public class Empleado extends Persona { private double salario; @Override public void presentarse() { System.out.println(nombre + ", empleado"); } }
Relación "es un"
Empleado ES UNA Persona

Agregación

public class Empresa { private String nombre; private List<Empleado> empleados; public Empresa(String nombre) { this.nombre = nombre; empleados = new ArrayList<>(); } public void agregarEmpleado(Empleado e) { empleados.add(e); } }
Relación "tiene un"
Empresa TIENE Empleados

7. Clases y Métodos Abstractos

public abstract class Figura { protected String color; // Método abstracto public abstract double calcularArea(); // Método concreto public void mostrarColor() { System.out.println(color); } } public class Circulo extends Figura { private double radio; @Override public double calcularArea() { return Math.PI * radio * radio; } } public class Rectangulo extends Figura { private double base, altura; @Override public double calcularArea() { return base * altura; } }
Clase Abstracta:

• No se puede instanciar
• Define plantilla
• Métodos sin implementar
• Subclases deben implementar

8. Modificador Final

Variables Final

public class Constantes { public static final double PI = 3.14159; private final String id; public Constantes(String id) { this.id = id; } }
Valores constantes que no cambian

Métodos y Clases Final

public class Padre { // No se puede sobrescribir public final void metodoFinal() { System.out.println("Final"); } } // No se puede heredar public final class Matematicas { public static int sumar(int a, int b) { return a + b; } }
Previene modificación y herencia

9. Interfaces

public interface Volador { void volar(); void aterrizar(); } public interface Nadador { void nadar(); } public class Pato implements Volador, Nadador { @Override public void volar() { System.out.println("El pato vuela"); } @Override public void aterrizar() { System.out.println("El pato aterriza"); } @Override public void nadar() { System.out.println("El pato nada"); } }
Contratos:

10. Manejo de Excepciones

public class Division { public static int dividir(int a, int b) throws ArithmeticException { if(b == 0) { throw new ArithmeticException("División por cero"); } return a / b; } public static void main(String[] args) { try { int resultado = dividir(10, 0); System.out.println(resultado); } catch(ArithmeticException e) { System.out.println("Error: " + e.getMessage()); } finally { System.out.println("Fin"); } } } // Excepción personalizada public class EdadInvalidaException extends Exception { public EdadInvalidaException(String msg) { super(msg); } }
Gestión de errores:

try
Código a probar

catch
Captura error

finally
Siempre se ejecuta

throws
Declara excepción

11. Hilos (Threads)

Extender Thread

public class MiHilo extends Thread { private String nombre; public MiHilo(String nombre) { this.nombre = nombre; } @Override public void run() { for(int i=1; i<=5; i++) { System.out.println(nombre + ": " + i); try { Thread.sleep(1000); } catch(InterruptedException e) {} } } } // Uso MiHilo h1 = new MiHilo("Hilo-1"); h1.start();

Implementar Runnable

public class Tarea implements Runnable { private String nombre; public Tarea(String nombre) { this.nombre = nombre; } @Override public void run() { System.out.println("Ejecutando: " + nombre); } } // Uso Thread hilo = new Thread(new Tarea("Tarea-1")); hilo.start();
Concurrencia: Los hilos permiten ejecutar múltiples tareas simultáneamente. Útil para operaciones en segundo plano, interfaces responsivas y procesamiento paralelo.

2. Estructuras de Control - Ciclos (for, while)

Ciclo for

continue: salta iteración

3. Listas (Arrays Dinámicos)

# Crear listas numeros = [1, 2, 3, 4, 5] frutas = ["manzana", "banana", "naranja"] mixta = [1, "texto", True, 3.14] # Acceder a elementos print(frutas[0]) # "manzana" print(frutas[-1]) # "naranja" (último) # Modificar elementos frutas[1] = "pera" # Agregar elementos frutas.append("uva") # Al final frutas.insert(0, "fresa") # En posición # Eliminar elementos frutas.remove("pera") # Por valor del frutas[0] # Por índice ultimo = frutas.pop() # Elimina y retorna último # Operaciones útiles print(len(frutas)) # Longitud print("manzana" in frutas) # True/False # Slicing (rebanado) numeros = [0, 1, 2, 3, 4, 5] print(numeros[1:4]) # [1, 2, 3] print(numeros[:3]) # [0, 1, 2] print(numeros[3:]) # [3, 4, 5] # Métodos útiles numeros.sort() # Ordenar numeros.reverse() # Invertir total = sum(numeros) # Suma
Listas:

• Colección ordenada
• Modificable (mutable)
• Permite duplicados
• Acceso por índice
• Tamaño dinámico

Características:
• Índices desde 0
• Índices negativos desde final
• Slicing flexible

4. Tuplas (Secuencias Inmutables)

# Crear tuplas coordenadas = (10, 20) persona = ("Juan", 25, "México") un_elemento = (5,) # Coma necesaria # Acceder a elementos print(coordenadas[0]) # 10 print(persona[-1]) # "México" # Desempaquetado x, y = coordenadas nombre, edad, pais = persona print(x, y) # 10 20 # Operaciones print(len(persona)) # 3 print(25 in persona) # True # Concatenación tupla1 = (1, 2) tupla2 = (3, 4) resultado = tupla1 + tupla2 # (1, 2, 3, 4) # Repetición repetida = ("a",) * 3 # ("a", "a", "a") # Slicing numeros = (0, 1, 2, 3, 4) print(numeros[1:3]) # (1, 2) # Métodos lista = (1, 2, 3, 2, 2) print(lista.count(2)) # 3 (veces que aparece) print(lista.index(3)) # 2 (posición)
Tuplas:

• Colección ordenada
• Inmutable (no se modifica)
• Permite duplicados
• Más rápidas que listas
• Usan menos memoria

Cuándo usar:
• Datos que no cambian
• Coordenadas (x, y)
• Valores de retorno
• Claves de diccionario

5. Diccionarios (Pares Clave-Valor)

# Crear diccionarios persona = { "nombre": "Juan", "edad": 25, "ciudad": "México" } estudiante = dict(nombre="Ana", edad=20) # Acceder a valores print(persona["nombre"]) # "Juan" print(persona.get("edad")) # 25 print(persona.get("tel", "N/A")) # Valor por defecto # Modificar valores persona["edad"] = 26 # Agregar elementos persona["email"] = "juan@email.com" # Eliminar elementos del persona["ciudad"] email = persona.pop("email") # Verificar existencia if "nombre" in persona: print("Nombre existe") # Obtener claves, valores, items claves = persona.keys() valores = persona.values() items = persona.items() # Iterar sobre diccionario for clave in persona: print(clave, persona[clave]) for clave, valor in persona.items(): print(f"{clave}: {valor}") # Actualizar múltiples valores persona.update({"edad": 27, "pais": "México"}) # Diccionario vacío vacio = {} datos = dict() # Diccionarios anidados escuela = { "estudiante1": {"nombre": "Ana", "edad": 20}, "estudiante2": {"nombre": "Luis", "edad": 22} }
Diccionarios:

• Pares clave-valor
• Modificable (mutable)
• Sin orden específico
• Claves únicas
• Acceso rápido O(1)

Características:
• Claves deben ser inmutables
• Valores pueden ser cualquier tipo
• Muy eficientes

Cuándo usar:
• Búsquedas rápidas
• Datos estructurados
• Configuraciones
• Bases de datos en memoria

6. Comparación de Estructuras

¿Cuándo usar cada una?

Listas: Cuando necesitas una colección ordenada y modificable. Ideal para datos que cambian frecuentemente.

Tuplas: Cuando tus datos no deben cambiar. Más eficientes en memoria y rendimiento. Perfectas para datos constantes.

Diccionarios: Cuando necesitas asociar claves con valores. Excelente para búsquedas rápidas y datos estructurados.

# Ejemplo comparativo lista = [1, 2, 3] # Modificable, ordenada tupla = (1, 2, 3) # Inmutable, ordenada diccionario = {"a": 1, "b": 2} # Clave-valor, no ordenado
